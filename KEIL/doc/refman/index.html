Introduction

This document describes the Freescale's software drivers for Kinetis L family of microcontrollers. These software drivers are provided in a source code form. The source code files are intended to be included directly into the application project or built separately into as a statically linked library. This document should give you all information about drivers' configuration and its use in your application. The Directory Structure explains the source files and header files important for your application. Please read the License document for terms and conditions under which you can use the software.

The Kinetis L series MCUs are the worlds most energy efficient, and bring new design possibilitiesto entry-level applications that have previously been limited by 8/16-bit MCU capabilities. L series MCUs combine the enhanced energy efficiency, performance and debug features of the new 32-bit ARM Cortex-M0+ core, with the scalability, feature integration and enablement of the Kinetis MCU portfolio. Available with multiple flash memory, analog, connectivity and HMI peripheral options, L series MCUs are hardware and software compatible with the Kinetis K series of MCUs based on the ARM Cortex-M4 core.

BASIC DATA TYPES
Freescale's Kinetis L Peripheral Drivers defines some basic data types to support code portability between different hardware architectures and tools. These Basic Data Types are defined in the C header file common.h.

ARM CORTEX-M0+ MACROS
Some architecture dependent routines and macros which provide interface to the ARM Cortex-M0+ core architecture are defined in the arm_cm0.h header file. Refer to Architecture Dependent Macros and Functions.
