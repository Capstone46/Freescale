Detailed Description

This section specifies default configuration structures for ADC module. Note that application specific configuration structure shall be created by the user and stored in the appconfig.h file. The easiest way to create user specific configuration structure is to copy default structure into appconfig.h file, rename it and update peripheral behavior by updating SET() and CLR() macros on each register field.

#define ADC_FAST_CONFIG (   nbit,
  trigsel,
  vrefsel 
)

Selects n-bit fast mode with optional trigger and reference selection.

  • Normal power config (MUXb)
  • Input clock is BUSCLK
  • Clock divider set to 2 (ADCLK = BUSCLK/2, considered as 12MHz max. ADCCLK)
  • fast = short sample time, high speed sequence, shortest sample time
  • optional trigger selection
  • ADxxa channels selected
  • internal voltage reference
  • one conversion
    Parameters
    nbit- number of bits to be converted (ADC mode option (number of converted bits))
    trigsel- SWTRG or HWTRG
    vrefsel- IREF or XREF

#define ADC_PRECISE_CONFIG (   nbit,
  trigsel,
  vrefsel,
  avgsampl 
)

Selects n-bit precise mode with optional trigger and ref. selection.

  • Normal power config (MUXa)
  • Input clock is BUSCLK
  • Clock divider set to 2 (ADCLK = BUSCLK/8)
  • precise = long sample time, high speed sequence, longest sample time
  • SW trigger
  • ADxxa channels selected
  • internal voltage reference
  • one conversion (also include set of conversions when AVG enable)
  • hardware avegaring enabled
    Parameters
    nbit- number of bits to be converted (ADC mode option (number of converted bits))
    trigsel- SWTRG or HWTRG
    vrefsel- IREF or XREF
    avgsampl- number of samples to be averaged (ADC Hardware Averaging Options)
